Visitor walkthrough

  1. Reset the scenario to a deterministic sample week.
  2. Point out the locked calendar events, task priorities, and hard deadlines.
  3. Generate the schedule and show which tasks fit, which do not, and why.
Next steps

The next product step is to polish the UI and UX so the planning state, tradeoffs, and schedule changes are easier to understand at a glance.

After that, the planner should support a stronger task model: a better to-do list, the ability to break work into smaller chunks, and richer sequencing controls.

The practical integration layer would then connect Zeit to tools people already use, including Google Calendar, Slack, and other workplace systems.

Longer term, the system should learn user patterns and preferences from repeated behavior, then adapt scheduling decisions around those habits. A fuller user system would sit underneath that so preferences, accounts, and personalized planning rules can persist across sessions.

Known limits

This version still cannot break tasks into smaller blocks, and the working window chooser applies to the current planning run rather than persisting as a user preference.
